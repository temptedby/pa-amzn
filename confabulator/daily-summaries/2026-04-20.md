# PA-AMZN — 2026-04-20

First full build day. Started with an empty scaffold, ended with a password-protected dashboard that can authenticate to Amazon SP-API, sync FBA inventory + Amazon's restock recommendations, and create real inbound shipment plans.

## What shipped today

### Phase 1 — Rule engines (pure logic, DB, tests)
- **Schema** (`src/lib/db/schema.sql`): campaigns, ad_groups, keywords, hourly_snapshots, bid_changes, search_terms, inventory, alerts, auth_sessions, prep_contacts, shipment_templates, shipments. Money stored as INTEGER cents; timestamps ISO-8601 UTC.
- **libsql client** (`src/lib/db/client.ts`) — same API for local SQLite + Turso prod. Migrations runner (additive ALTER TABLE strategy for column adds on existing DBs).
- **Momentum bid engine** (`src/lib/rules/bid-engine.ts`) — pure function:
  - 7d spend ≥ $4 with 0 conversions → kill switch
  - 0 impressions + bid < $2 → bump +10% / bid ≥ $2 → pause
  - 0 conversions under spend threshold → −10% exploration drop
  - Above soft cap (default $2 single-pack, scales w/ list price): default is decrease unless 3d ACOS ≤ aspirational (10%)
  - 3d vs prior-3d ACOS comparison for momentum direction
  - 6-hour rate limit per keyword
- **Harvest engine** (`src/lib/rules/harvest-engine.ts`) — 2+ orders, <30% ACOS → phrase+exact; <50% → exact only; emits negative-exact suggestions for source campaign.
- **38 tests passing** (16 bid, 8 harvest, 3 sp-api, 4 inventory, 7 restock).

### Phase 2 — Dashboard + auth

### Phase 3 — SP-API integration
- **SP-API client** (`src/lib/amazon/sp-api.ts`) — LWA refresh→access token with 1 min pre-expiry cache, bearer-auth request wrapper. Captures non-2xx response bodies in `SpApiError.body` + logs them.
- **Credentials form** at `/settings` — writes to `.env.local` with 0600 perms, masked display. Dev-only; production uses Vercel env vars.
- **FBA inventory sync** — `/fba/inventory/v1/summaries` paginated fetch, DB upsert.
- **Inventory page**: live DB data, inline-editable thresholds (press Enter to save), hyperlinks to Seller Central edit + public amazon.com listing, sort by FBA qty desc, active/inactive filter, sync result banner.
- **Reports API client** (`src/lib/amazon/reports.ts`) — create → poll → download → gunzip. Generic.
- **Restock recommendations** — pulls `GET_RESTOCK_INVENTORY_RECOMMENDATIONS_REPORT`, parses TSV, updates inventory table with Amazon's "send N by date Y" suggestions. Shows in new column.
- **Prep contacts CRUD** at `/settings`.
- **Shipment templates** per-SKU on Inventory page (carton dims, weight, prep contact, notes). Most empty for now.
- **Shipments: list + new + detail pages** with status pills.
- **`createInboundPlan`** (v2024-03-20) — end-to-end call works. Validated with real SP-API creds: SUCCESS response for shipment #3 (plan `wf719bbd32…`).

## Strategy + research decisions

- **Bid strategy**: The momentum-based hourly engine is what we're shipping; the static-bracket PRD rules are obsolete. Key logic: ride winners up while ACOS stays strong, drop hard when it weakens, $4 spend-no-conv kill switch. Market-share capture while ACOS favorable, not pure profit defense. Saved to project memory.
- **Inbound label automation** — earlier research said skip (ROI too low). Reversed when user clarified Megan's time on inventory + label coordination is hours/month, not minutes. Now on the "finish it" list.
- **Ads API** is a **separate OAuth from SP-API**. One LWA refresh token does NOT work for both. Registration still pending at advertising.amazon.com.
- **`prepOwner: "SELLER"` is rejected** for simple products (phone clips). Must send `"NONE"`. Fixed.
